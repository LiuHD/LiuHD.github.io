note-about-database-note

发表于 2019-01-27

这篇文章主要介绍数据库中读操作的几种异常现象，数据库不同纬度锁的分类，数据库的隔离级别等

几种数据库读的异常状况

脏读

这种情况发生在两个事务并发执行时，A事务在进行读取时读到了B事务尚未提交的数据，这种数据有可能会被回滚，变得无效，是脏的，所以称为脏读

A事务 B事务
select * from users where id = 1
results: 6
update users set age = 7 where id = 1
select * from users where id = 1
results: 7

不可重复读

这种情况下是A事务进行了读取后，B事务对数据进行了修改，然后A想验证自己读的数据，再次读取时，读取了B更新后的数据，与自己开始时读取的数据不一致，所以成为不可重复读
| A事务 | B事务 |
| ————————————————- | ————————————— |
| select * from users where id = 1
results: 6 |
| | update users set age = 7 where id = 1 |
| select * from users where id = 1
results: 7 |

幻读

幻读可以称之为表级别的不可重复读，当A事务进行了读取后（假设这种读取是表级别的，比如count），B事务对数据进行了更新（假设是更新，插入，删除这种操作，会影响A结果的那种），之后A事务想验证自己读的数据，所以再次读取，返现读取到的数据由于B的修改发生了变化，所以就好像自己“幻”读了

为了解决以上的读问题，需要引入隔离级别的概念，隔离级别是针对事务而言，事务具有ACID属性，依次是原子性，持久性，一致性，隔离性，搞的隔离级别可以杜绝一些读的异常状况，但是，太高的隔离级别也会降低数据库的处理速度，因为更高的隔离级别以为着更频繁的加锁及冲突，等待，所以根据不同的业务类型选择相应的隔离级别尤为重要。

隔离级别

隔离级别的一般划分，级别从高到低依次是：

  • 可序列化
  • 可重复读
  • 提交读
  • 未提交读

未提交读下上述的三种异常情况都可能会发生

读：不加锁
写：加行级共享锁，读语句执行完立即释放

提交读，仅可以避免脏读

读：加行级共享锁，读语句执行完立即释放
写：加行级排他锁，事务结束时释放

可重复读，可以避免脏读与不可重复读

读：加行级共享锁，事务结束时释放
写：加行级排他锁，事务结束时释放

可序列化，可以避免幻读

读：加表级共享锁，事务结束时释放
写：加表级排他锁，事务结束时释放

锁的分类

按粒度大小（从上到下粒度变大，加锁开销变大）

  • 行级锁(又分为共享锁与排他锁)
    开销大，加锁慢；会出现死锁；锁定粒度最小，发生锁冲突的概率最低，并发度也最高。innoDB引擎支持
  • 页级锁
    开销和加锁时间界于表锁和行锁之间；会出现死锁；锁定粒度界于表锁和行锁之间，并发度一般，仅BDB引擎支持
  • 表级锁(又分为共享锁与排他锁)
    开销小，加锁快；不会出现死锁；锁定粒度大，发出锁冲突的概率最高，并发度最低。MYISAM与INNODB都支持

delegate 绑定事件的冒泡拦截问题

发表于 2017-12-15

对于一个静态的元素(在页面完成加载时已经存在，非后期动态生成)，如果用delegate的话，会出现无法阻止该事件冒泡的情况，具体的就是delegate方式绑定事件本身的工作方式就是把事件绑定在父元素上，然后在事件产生的时候，逐级冒泡到该父元素，父元素才会触发这个事件，请注意这个事件在触发的时候已经是逐级冒泡之后的事了，所以在事件处理函数里拦截什么的都不会起作用。

网页邮件编写注意事项

发表于 2017-12-14

1、不能使用任何形式的背景图片；
2、样式要写到HTML代码中，类似于style=”text-decoration:none; color:#666; font-size:12px;” ,另外<’style><’/style>中的样式 必须放在body中才能起作用，但是在转发的过程中部分样式会丢失，所以不建议使用
3、在每个图片都要加上alt和title
4、要考虑邮件的宽度，很多用户是在用笔记本，笔记本的分辨率是1280*708,。。。所以为了第一眼的良好用户体验，必须有合适的宽度
5、所有图片都必须用绝对地址，否则在客户终端不会显示，在foxmail中可以显示；
6、在布局上，用table，不用div，因为在邮箱中连margin:0 auto都不起作用，css属性没有全部测试，但是真的很受限制，那么就用table吧；
7、尽量每个块都要加上宽度，不写宽度，会有莫名奇妙的bug；
8、显示效果以邮箱为准，有些情况浏览器渲染的页面效果和邮箱渲染的不一样；
